## Objectif

Utiliser les outils de profiling Linux (perf) pour identifier les
goulots d'étranglement des versions ssgemm, puis concevoir une version
vectorisée AVX 256 bits (mcblas010).

## Contribution principale : mcblas010 (Q-4.1)

La clé de la vectorisation est de placer c en boucle interne (stride-1
sur B et C) et d'extraire A(r,d) comme invariant mis en registre.
Le compilateur génère alors automatiquement des instructions AVX 256 bits
(vmulps, vaddps), confirmé par -fopt-info-vec-optimized :

    mcblas3.cpp:31: optimized: loop vectorized using 32 byte vectors

Taille de bloc optimale : T=64 (49 152 octets < L1d=64 KiB avec AVX).

## Observation

v010-O2 est meilleur que v010-O3 sur cette machine. Le niveau -O3
interfère avec la vectorisation guidée manuellement en ajoutant un
déroulage automatique supplémentaire contre-productif.

# Compilation
Créer un répertoire "build", se placer dans ce répertoire et exécuter la commande cmake suivante :
        =cmake ../=

* Exécution
L'exécution des tests est maintenant réalisée directement par la commande make
Elle peut être appelé par la commande :
        =make RALL=

* Génération des données pour les graphiques et des graphiques
        =make GALL=

* Profiling
Pour utiliser les outils de profiling, modifier le fichier :
        =tests/runs/CMakeList.txt=
